# Jimy’s Portfolio

A personal portfolio built with **Vue 3**, **Vite**, and **Tailwind CSS**.

This is not meant to be a plain developer profile page.  
I wanted it to feel like a small digital space with its own mood: calm, playful, cinematic, a little scrapbook-ish, and still readable enough for real people to actually explore.

Live site: [borgirrax.github.io/portofolio](https://borgirrax.github.io/portofolio/)

---

## What this is

This portfolio is my personal space to show:

- frontend projects I have built,
- landing page and UI exploration,
- visual storytelling experiments,
- a bit of personality beyond just “here are my skills”.

The direction is simple:  
**make a portfolio that feels human, not corporate.**

The site leans into big typography, blue-gray tones, analog grain, handwritten notes, subtle doodles, and bento-style project cards. The goal is to make the page feel like one continuous cinematic scrapbook, not a pile of disconnected sections.

## Design direction

The visual direction is built around a few rules:

- big cinematic typography,
- soft blue-gray palette,
- analog grain texture,
- subtle dotted grid background,
- handwritten annotations,
- minimal futuristic scrapbook mood,
- generous spacing,
- calm transitions,
- no overkill neon, no excessive particles, no visual chaos.

The design tries to stay playful without becoming childish, and polished without becoming too corporate.

## Running locally

Clone the repository:

```bash
git clone https://github.com/BorgirRaX/portofolio.git
cd portofolio
```

Install dependencies:

```bash
npm install
```

Start the development server:

```bash
npm run dev
```

Build for production:

```bash
npm run build
```

Preview the production build locally:

```bash
npm run preview
```

---

## Deployment note

This project is deployed using **GitHub Pages**.

Because the repository is named `portofolio`, the Vite base path needs to match the GitHub Pages route:

```js
base: '/portofolio/'
```

For Vercel or a future custom domain, the base path can be changed back to `/`.
